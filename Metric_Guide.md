# คู่มือการอ่านค่าผลการเทรน YOLO (Training Metrics Guide)

เอกสารนี้สรุปความหมายของตัวเลขต่างๆ ที่ปรากฏในหน้าจอขณะเทรนและหลังเทรนเสร็จครับ

---

## 1. ช่วงกำลังเทรน (Training Interval)
บรรทัดนี้บอกสถานะว่าโมเดล "เรียนรู้" ได้ดีแค่ไหนในรอบนั้นๆ

```text
Epoch    GPU_mem   box_loss   seg_loss   cls_loss   dfl_loss  Instances       Size
 8/20      15.7G      1.903        2.3      1.393      1.268        406        640
```

| ค่า (Metric) | ความหมาย | หลักการจำ |
| :--- | :--- | :--- |
| **Epoch** | รอบการเทรน (เช่น 8/20 = รอบที่ 8 จาก 20) | ยิ่งเยอะ ยิ่งเรียนรู้นาน |
| **GPU_mem** | การ์ดจอถูกใช้ไปเท่าไหร่ | ถ้าเต็ม = Error (ลด batch size ช่วยได้) |
| **box_loss** | ความเพี้ยนของ "กรอบสี่เหลี่ยม" (ตีเบี้ยวไหม?) | **ยิ่งน้อย ยิ่งดี (↘)** |
| **Instances** | จำนวนวัตถุที่เจอในรอบนี้ (Batch) | เป็นแค่ข้อมูลประกอบ ไม่ใช่คะแนน |
| **seg_loss** | ความเพี้ยนของ "การระบายสี" (ระบายเกินไหม?) | **ยิ่งน้อย ยิ่งดี (↘)** |
| **cls_loss** | ความเพี้ยนของ "ประเภทวัตถุ" (ทายผิดว่าเป็นอย่างอื่นไหม?) | **ยิ่งน้อย ยิ่งดี (↘)** |

> **สรุป:** ถ้าค่า Loss ต่างๆ **"ลดลง"** เรื่อยๆ แปลว่าโมเดลกำลังฉลาดขึ้นครับ

## 2. ช่วงตรวจสอบผล (Validation Interval)
บรรทัดนี้คือ "คะแนนสอบ" บอกว่าโมเดลเก่งจริงไหมเมื่อเจอข้อสอบ (รูป) ที่ไม่เคยเห็น

```text
Class     Images  Instances      Box(P          R      mAP50  mAP50-95)
  all          8       1528      0.442      0.531      0.401      0.196
```

| ค่า (Metric) | ชื่อเต็ม | ความหมายภาษาคน | เป้าหมาย |
| :--- | :--- | :--- | :--- |
| **Instances** | **Instances** (จำนวนวัตถุ) | จำนวนวัตถุทั้งหมดที่มีในภาพทดสอบชุดนี้ | - |
| **P** | **Precision** (ความแม่นยำ) | "ที่ทายมาเนี่ย...ถูกกี่ตัว?"<br>*(ทายมั่วจะตก ทายชัวร์จะสูง)* | **ยิ่งมาก ยิ่งดี (↗)** max = 1.0 |
| **R** | **Recall** (ความครบถ้วน) | "ที่มีอยู่จริงเนี่ย...หาเจอกี่ตัว?"<br>*(หาไม่ครบ คะแนนจะตก)* | **ยิ่งมาก ยิ่งดี (↗)** max = 1.0 |
| **mAP50** | mean Average Precision @ IoU 0.5 | **"เกรดเฉลี่ยรวม"** (เกณฑ์มาตรฐาน)<br>*ดูค่านี้เป็นหลักเพื่อวัดความเก่ง* | **ยิ่งมาก ยิ่งดี (↗)** |
| **mAP50-95** | mAP @ IoU 0.5-0.95 | **"เกรดเฉลี่ยรวม"** (เกณฑ์โหดหิน)<br>*ต้องตีกรอบเป๊ะมากๆ ถึงจะได้คะแนนนี้* | **ยิ่งมาก ยิ่งดี (↗)** |

## 3. วิธีวิเคราะห์ปัญหา (Troubleshooting)

*   **P สูง แต่ R ต่ำ:** โมเดล "ขี้อาย" คือทายเฉพาะตัวที่มั่นใจมากๆ ตัวที่ซ่อนอยู่ไม่กล้าทาย (หาไม่ครบ)
*   **P ต่ำ แต่ R สูง:** โมเดล "ขี้โม้" คือทายกราดไปหมด เจอจริงบ้าง ไม่จริงบ้าง (หาครบแต่ขยะเยอะ)
*   **P และ R ต่ำทั้งคู่:** โมเดล "ยังไม่ฉลาด" (เทรนน้อยไป หรือ ข้อมูลสอนแย่)
*   **Loss ลด แต่ mAP ไม่เพิ่ม:** เรียกว่า **Overfitting** (ท่องจำข้อสอบเก่า แต่ทำข้อสอบใหม่ไม่ได้) -> ควรหยุดเทรน
